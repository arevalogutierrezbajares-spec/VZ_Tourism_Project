## Priority Issues

### P0 (broken / critical)

1. **Secret key exposure** — `.env.local` contains live Anthropic, Groq, WhatsApp, and Supabase service-role keys in plaintext. Verify `.gitignore` covers this file; rotate all keys if any commit history contains them.

2. **Test suite broken (10 suites, 57 failures)** — `createServiceClient is not a function` in integration tests. The mock for `lib/supabase/server.ts` is not properly wiring `createServiceClient`. Fix the jest mock in `__tests__/` setup to export the function correctly.

3. **`loadDrivers` / `loadVehicles` accessed before declaration** (`app/(ruta)/ruta/dispatch/drivers/page.tsx:31`, `vehicles/page.tsx:29`) — ESLint `react-hooks/immutability` error. Functions declared with `async function` are hoisted in JS but the React Compiler rejects this pattern as it prevents stable references. Convert to `useCallback` or move declarations above the `useEffect`.

### P1 (high priority)

4. **`Math.random()` in render** (`app/onboard/[slug]/OnboardingWizard.tsx:1548–1553`) — generates new random values every re-render, causing visual flicker in the confetti animation. Pre-compute values in `useMemo` or `useState`.

5. **`react-hooks/set-state-in-effect` (15 files)** — calling `setState` synchronously at the top of a `useEffect` body is flagged by the React Compiler. Most cases are legitimate (mounting flags, loading triggers) but should be structured correctly (e.g., run effect logic then setState in a callback, or use `useLayoutEffect` where appropriate).

6. **`any` in Stripe webhook handler** (`app/api/stripe/webhooks/route.ts` — 6 occurrences) — Stripe events should be typed with `Stripe.Event` and discriminated union subtypes.

7. **Missing error boundaries for critical routes** — RUTA dispatch, PMS pages, onboarding wizard, and auth pages all lack `error.tsx`. An unhandled error in any of these crashes the full route segment silently.

### P2 (medium priority)

8. **`@typescript-eslint/no-explicit-any` in auth pages** — `login`, `register`, `provider-register` all cast form errors as `any`. Use `unknown` + type guard or type from React Hook Form's error types.

9. **`react/no-unescaped-entities` (13 errors)** — Apostrophes and quotes in JSX text strings need escaping (`&apos;`, `&ldquo;` etc.). Mechanical fix but required for lint to pass.

10. **`@next/next/no-img-element` (~35 warnings)** — Replace `<img>` with `next/image` throughout listing cards, admin pages, and discover pages. LCP performance impact.

11. **`jsx-a11y` violations** — `AvailabilityCalendar.tsx:158` (`aria-selected` on button role) and `LocationInput.tsx:141` (combobox missing required ARIA) are accessibility errors, not cosmetic.

12. **`react-hooks/exhaustive-deps` (~12 warnings)** — Missing dependencies in useEffect/useCallback arrays risk stale closures.

### P3 (low / backlog)

13. **Unused imports/variables (~80 warnings)** — Systematic cleanup pass needed. Worst files: `AiSettingsPanel.tsx`, admin providers page, OnboardingWizard.

14. **`eslint` scanning build artefacts** — `frontend/.next/` is included in lint scope, polluting output with thousands of irrelevant violations. Add `frontend/.next` and `frontend/node_modules` to `.eslintignore` or ESLint `ignorePatterns`.

15. **`console.log` in `expire-zelle` cron route** — Remove before production.

16. **7 open TODO comments** in payout, provider-settings, and webhook routes — track as issues.

17. **No test coverage metric** — once test suite is fixed, configure `jest --coverage` thresholds in `jest.config.js` (target ≥60% for P1).
